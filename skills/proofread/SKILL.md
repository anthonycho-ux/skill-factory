# proofread — AI Draft Polish

## Trigger

Invoke when:
- User says "proofread", "proofread this", "polish this", "clean this up"
- Text was generated by a non-native or AI model (Minimax, Claude, Gemini, etc.)
- Any time human-facing writing needs a pass before publishing

## What It Does

Takes raw AI-generated text and polishes it to human taste. Not rewriting — cleaning up the tells that make readers feel the seams:

- **Typography**: em-dashes, en-dashes, curly quotes, ellipses — consistency and correctness
- **Structural**: parallel structure, missing words, orphaned fragments
- **Cultural**: Chinese-origin characters, UTF-8 artifacts, unnatural phrasing
- **Rhythm**: sentences that run too long, comma splices, buried clauses

## The Insight

Different models have different blind spots. Minimax M2.7 leaves subtle Chinese-character artifacts and rhythm issues. Claude produces clean grammar but sometimes flat structure. No single model is perfectly aligned to your audience's taste.

The move: use a local, fast model (Gemma) as a taste filter on other AI output. Gemma catches what the originating model missed — because it's reading for a different audience, with different expectations.

## Workflow

```
1. Identify the text to proofread (user provides file, path, or pasted content)
2. Load into Gemma E2B via Ollama (or best local model available)
3. Send with this system prompt (or equivalent):

   "Proofread the following text. Fix grammar, punctuation, typography,
   and clarity issues only. Do not rewrite or restructure. Preserve the
   author's voice. Return only the corrected version."

4. Extract Gemma's response
5. Diff the original vs corrected — list specific changes made
6. Present diff to user for approval
7. On approval — save corrected version, overwrite or suffix "_proofread.md"
```

## Rules

- **Preserve voice.** If the text sounds like the user, don't change the voice. Fix the mechanics, not the personality.
- **No re-writing.** Don't add content, restructure arguments, or improve the writing. Only clean the seams.
- **Show the diff.** Always list what changed — this is how the user calibrates the model's taste against their own.
- **Name the originating model.** Note what generated the original text. Different models have different tells — this helps you tune the skill over time.
- **Local first.** Use Ollama (Gemma E2B or best available). No API calls for proofreading — this is a fast, local task.

## Examples

**Input** (Minimax M2.7 output):
```
Most AI skills are magic spells: you say the right words, something happens.
```

**Gemma corrects to:**
```
Most AI skills are magic spells: you say the right words, and something happens.
```

**Diff reported:**
- Added missing "and" — sentence was a fragment without conjunction

---

**Input** (with UTF-8 artifact):
```
...decades-old findings from cognitive science. And almost nobody in the AI tooling world was building with them.
```

**Gemma corrects to:**
```
...decades-old findings from cognitive science. And almost nobody in the AI tooling world was building with them.
```

**Diff reported:**
- No changes needed — confirmed clean

---

**Input** (structural mess):
```
I'd re-solve the same problem. Miss the same insight. Arrive at the same dead end I'd visited three months ago and completely forgotten.
```

**Gemma corrects to:**
```
I'd re-solve the same problem, miss the same insight, and arrive at the same dead end I'd visited three months ago and completely forgotten.
```

**Diff reported:**
- Collapsed three short sentences into parallel structure with commas and "and"
- Maintained the rhythm and voice

## Dependencies

- Ollama running locally (`ollama serve`)
- Gemma E2B or equivalent model loaded (`ollama pull gemma4:e2b` if missing)

## Calibration

Track what Gemma catches vs what a human would catch:

- If Gemma misses something the user fixes manually → add to system prompt
- If Gemma changes something the user restores → add to "do not touch" list
- If a different model becomes available that catches more → swap in `model` field

This skill compounds. The more you use it, the better it reads for your taste.
