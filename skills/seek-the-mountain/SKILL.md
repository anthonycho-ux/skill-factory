# /seek-the-mountain — Choose the Version That Creates Legends

## What It Is

Before committing to a problem, a partner, or a path — identify the easy version and deliberately choose the hard one. The easy version is almost always the wrong answer.

The principle: legends are forged by fighting legends, not by running through no-name opponents. The version of the challenge that will be remembered is almost never the comfortable one.

---

## Trigger

Invoke when:
- Choosing between multiple versions of a problem (easy vs. hard opponent, safe vs. contested market, incremental vs. moonshot)
- A team is defaulting to the comfortable version of a challenge
- "Nobody else could do it" is a product description, not a warning
- Keywords: "play it safe", "start small", "build up to", "ease into", "not ready for the big one"

---

## Steps

### 1. What is the easy version?

Name it specifically. "We'll start with SMB customers." "We'll partner with the safe incumbent." "We'll enter after someone else proves it works." The easy version always sounds reasonable.

### 2. What is the mountain?

The version where winning matters. Where if you pull it off, people will remember it. Where if you fail, you fail loudly and with dignity.

### 3. Is this an ant hill or a mountain?

*"Is this the mountain, or the ant hill?"* If the outcome doesn't matter if you win — if no legend is possible in the success — the mountain is elsewhere.

### 4. Can you reach the mountain?

Capacity check: do you have the people, time, capital to attempt it? If not: build toward it, find partners who can climb it with you, or accept the ant hill honestly — don't dress it up as a mountain.

### 5. The dinner party test.

Would you tell the story of the easy version in 10 years? If not — don't do it.

---

## Decision

```
Mountain exists + capacity exists → Go to the mountain.
Mountain exists + capacity doesn't → Build toward it. Find the right partners.
Mountain doesn't exist (ant hill) → Don't do it. Or do it so small it doesn't matter.
```

---

## Rules

- **The easy version is almost always wrong.** Not always — sometimes you need to build capacity first. But default to the easy version and you're building an ant hill.
- **The mountain is where legends are made.** Conor needed Khabib. Jon Jones needed Gustafsson.
- **"Nobody else could do it" is a product description.** If the problem is easy enough that anyone can solve it, the solution isn't worth much.
- **Institutional memory is built on mountains.** The stories people tell about you 20 years later are the mountains you climbed, not the ant hills you walked around.
