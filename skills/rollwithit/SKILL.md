# /rollwithit — Constraint Execution Skill

## What It Is

Operate when the plan meets reality — when the constraint appears, the thing breaks, the resource disappears, or the assumption was wrong.

Robert Rodriguez's method: you don't control the conditions, you control your response. Every disaster is raw material. The bar burning down looked better charred. The Steadicam going out of focus became the look. The constraint is not the obstacle — it is the creative brief.

---

## Trigger

Invoke when:
- User says "rollwithit", "this broke", "we don't have X", "stuck"
- A plan meets an unexpected obstacle mid-execution
- A resource was removed or never existed
- The original assumption was wrong and the work must continue anyway

---

## Inputs

- **The obstacle** — what broke, what's missing, what's constrained
- **The current state** — where you are in the work, what's already done
- **The deadline** — how much time remains

---

## Phase 1 — Absorb

1. Name the obstacle exactly. Not "things went wrong" — "the database is down, we have 3 hours, the demo is tomorrow."
2. State what you expected. This is not self-blame — it's calibration. What assumption did the obstacle invalidate?
3. State what you still have. Every constraint leaves something. Name it.

---

## Phase 2 — Convert

4. Ask: **Is the obstacle the brief?** Sometimes the constraint is telling you what to make. The burning bar wasn't a problem to solve — it was the set.
5. For each remaining resource or capability: what does this enable that the original plan didn't?
6. Write one sentence describing what this version of the work looks like. If you can't, the obstacle hasn't been converted yet.

---

## Phase 3 — Execute

7. Do the converted version. Not the original plan in reduced form — the thing this constraint actually makes possible.
8. Ship it. The constraint-executed version is almost always more interesting than the plan would have been.

---

## Success Criteria

- [ ] Obstacle named exactly
- [ ] Assumption that was invalidated stated
- [ ] What remains stated
- [ ] Converted brief written in one sentence
- [ ] Something shipped

---

## The Deeper Principle

**"I don't have time to be careful."** — Robert Rodriguez

The difference between a professional and an amateur is not skill — it's what happens when the plan breaks. The amateur stops. The professional converts and ships.

Rodriguez shot El Mariachi for $7,000. He didn't have money for dolly tracks — so the Steadicam shots became the look. He didn't have time for rehearsal — so the rawness became the style. He didn't have anything except a camera and a deadline. That was enough.

---

## Rules

- **Do not fight the constraint.** Fighting consumes time and produces nothing.
- **Name what you still have.** Every obstacle leaves options open.
- **The converted version is not a compromise.** It is the actual thing this set of constraints makes possible.
- **Ship it.** Waiting for the original conditions to return is not a strategy.
