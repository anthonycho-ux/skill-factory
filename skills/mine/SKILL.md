# /mine — Make It Sound Like Mine

## What It Is

A two-model stack that strips AI texture from generated text and reclaims the voice that is unmistakably human, unmistakably yours.

**Generate** (any model) → **Polish** (Gemma E4B) → **Approve** (you)

The goal is not perfect grammar. The goal is **soul** — the part of writing AI cannot fabricate.

---

## Trigger

Invoke when:
- User says "proofread", "polish", "clean this up", "make this sound human", "/mine"
- Text was generated by any AI model and needs a human pass
- Writing is about to be published or shared

---

## Inputs

- **Text** — file path, pasted content, or model output
- **Source model** — name the originating model (e.g. "Minimax M2.7", "Claude", "Gemini") for calibration

---

## Workflow

```
1. Identify text and name the source model
2. Call Ollama /api/chat (NOT /api/generate — gemma4:e2b silently
   truncates responses at high token counts):

   curl -s http://localhost:11434/api/chat -d '{
     "model": "gemma4:e4b",
     "messages": [
       {"role": "system", "content": "You are a taste filter. Read this text and fix only the things that make it sound AI-generated: typos, rhythm issues, structural awkwardness, character artifacts, flat parallel structure. Do NOT rewrite. Do NOT restructure arguments. Preserve the author'\''s voice — only remove the seams. Return only the corrected text."},
       {"role": "user", "content": "[USER TEXT]"}
     ],
     "stream": false
   }'

3. Extract response from .message.content in the JSON
4. Diff — list exactly what changed and why
5. Present diff to user for approval
6. On approval — save, overwrite, or suffix "_mine.md"
```

---

## Success Criteria

- Output sounds human, not AI-generated
- Voice preserved — personality not sanded
- Diff clearly shows what changed and what stayed

---

## Rules

- **Preserve soul.** Don't sand down personality to make it "correct." If the voice is strange in an interesting way, keep it.
- **Never rewrite.** This is a reclaiming pass, not an editing pass. Only remove what AI added — don't restructure, don't add, don't improve.
- **Show the diff.** Always list what changed. The diff is how taste calibrates over time.
- **Name the source model.** Every model has different tells — tracking the source sharpens Gemma's future passes.
- **Local first.** Ollama only, no API calls — fast, private, yours.
- **Light touch.** A human reads and adjusts. That's the skill, not rewriting.

---

## What Gets Fixed

- **Rhythm** — flat cadence, staccato fragments, buried clauses
- **Typography** — em-dashes, spacing, punctuation habits
- **Structure** — missing connectors, parallel sentences, orphaned fragments
- **Artifacts** — Chinese-origin character noise, UTF-8 artifacts, unnatural phrasing from the originating model

---

## Calibration

The skill compounds. Track it:

- **Gemma changes, user reverts** → add that pattern to "do not touch"
- **Gemma misses something** → add to the system prompt
- **New source model** → note its tells separately

Over time, Gemma learns your palate. The tool becomes an extension of your sensibility.

---

## Dependencies

- Ollama running locally (`ollama serve`)
- Gemma E4B loaded (`ollama pull gemma4:e4b` if missing)
- **Critical:** Use `/api/chat` — `/api/generate` silently drops gemma4 responses at high token counts

---

## The Deeper Intention

`/mine` is about the realm AI cannot touch: the specific rhythm of your thoughts, the way you phrase something no model would invent, the thing you're actually trying to say.

The skill removes what AI added. What remains is what you meant.
